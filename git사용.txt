<설명>
로컬1에서 하고 허브에 올리고 다시 로컬2에서 하고싶을때는 clone을 쓰는게 아니라 git pull을 사용하는것. 
즉 init과 clone은 한번만 하면 된다.

gitdesk에 파일-뉴리파짓->이름이랑 루트 작성- > 생성
그 해당하는 폴더에 파일을 넣으면 데스크에 자동으로 나온다. 
gitdesk는 변화된 내용을 말해주는 것.  -------- 이건 다 로컬에서만 일어나는 일. 


github에 올리려면 데스크에 publish republic(push) 을 눌러서 프라이버시표시 빼고 만듬. 
꼭 데스크를 사용해서 안해도 되고 깃허브 페이지에 직접가서 업로드를 사용하면 됨.
깃데스크는 버전관리까지 해주는 용도!


집에가서는 깃데스크- 파일 - 클론리파짓토리 - 파일선택후  루트 지정하고 클론. 하면 성공!
직접 깃허브페이지에서 파일 선택하고 수정하고 꼭 commit 설명해주고 데스트에가서 리파지토리 -> pull을 
누르면 그 파일이 수정된다. / 즉 집에가서는 처음만 클론하고, 계속 나머지는 .pull만하며 된다.
